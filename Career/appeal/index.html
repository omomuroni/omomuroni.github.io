    自己PR#

    これまでの職務経歴を交えて、自己紹介してください#

    もともとは新卒で飲食業界に務めておりましたが、IT業界へ転職し、現在は大手通信キャリアのコアネットワークの検証業務に 携わっております。IT業界での経験について、2社での現場経験があります。

    1社目のVSN株式会社、現在のAkkodis コンサルティング株式会社では、国内ネットワーク機器ベンダターであるアライドテレシス社へ就業しておりました。 アライドテレシス社では主にテクニカルカスタマーサポートエンジニアとして、自社製品に関する技術的な問合せ対応とHW障害のログ解析・交換対応業務を 経験いたしました。また、運用・監視業務もありましたので、企業ネットワークの設定変更や、定期レポートの作成、障害発生時の1次対応も経験いたしました。 ここでの経験につきまして、ベンダーコントロールにおいてとても役に経験となりましたあ。 アライドテレシス社での経験が、ベンダーコントロールにおいてとても役に経験でした。今の職場でベンダーに問い合わせる際、ベンダー目線でどんな情報が必要なのか？ といった視点で問い合わせることでその後の対応が円滑に行われるケースが多くありました。

    通常業務に加えて、 アライドテレシス社での業務の傍ら、CCNPの資格を取得し、よりエンジニアとして早期にスキルアップを目指したいと思い、 転職し、今の大手Slerにて就業し、通信キャリアネットワークの検証業務に携わっております。

    検証業務では、通信キャリア向けのハイエンドルーターを扱い、HW/ネットワークプロトコルの両観点からの動作確認検証と、 商用設備を模擬した大規模な検証環境を構築した上での障害試験・エージング試験などを経験致しました。

    また、検証業務だけではなく、python を用いた検証業務の自動化にも携わりました。検証自動化に携わる中で、 人の手で行われていた作業をコードに置き換えることで業務を効率化することができとてもやりがいを感じました。

    この経験からネットワークだけではなく、LinuxやPython などのネットワーク以外の知識も学び、継続して資格取得にも取り組みました。

    しかしながら、今の現場ではネットワーク以外の技術は特に必要とされず、検証の自動化の推進も滞っており、 今の自分のスキルを活かす場がないことに危機感を覚え、幅広く技術を身につけられ、それを活かすことができる環境で働きたいと考え転職活動に取り組んでおります。

    また、 延期のため案件として凍結となりましたが、Cisco Crosswork Network Controller を用いた 次世代SDNトランスポートの運用管理自動化の概念実証にも 携わった経験があります。概要だけご説明すると、大規模な通信キャリアのネットワーク設備に対し、Cisco 社の製品を用いてネットワーク運用・制御業務フローのソフトウェア化に関する案件でした。

    現在の職場でのやりがいは何ですか？#

    単純な作業を効率化することです。

    検証業務自体や、試験結果の確認・ログから結果の抽出など、自動化できる業務をまだまだ人の手で行っているためそれらを 自動化し、工数を削減することにやりがいを感じています。

    python を使ってログの解析を行ったり、router のコンフィグ作成と設定変更を自動化したり、実際にコードを書いて業務の効率化に貢献いたしました。

    転職を考えた理由は何ですか？#

    ・ネットワークだけではなく、サーバーやデータベース、プログラミングなどITに関して幅広い知識を身につけたいと考えたから →こう考えるきっかけとなったのが、検証業務の自動化に携わった経験からです。 それまで、人の手で行われていた作業をコードに置き換えることで業務を効率化することができとてもやりがいを感じました。 また、実際に自分が書いたコードが動く様子をみて何かを作る喜びを感じました。

    ネットワークの知識だけではなくプログラミングのスキルを組み合わせることで業務を効率化することができ、 この経験から技術を組み合わて価値を生むことに魅力を感じましたので、ネットワークの知識だけではなく、 幅広く技術を身につけられる環境で働きたいと考え転職活動に取り組んでおります。

    転職を考えた理由は、ネットワークエンジニアの経験を積んでいく中で、狭い領域の技術に限定されていることに対して危機感を覚えたからです。 私は常に新しい知識を学び、幅広いスキルを身につけることに興味があります。

    クラウドエンジニアとして働くことで、よりITに関する幅広い知識を身につけることができると考えております。

    クラウドはオンプレとは異なり、モノを用意する必要がなく、より本質的な技術に集中するとことができるかと思います。

    ネットワークエンジニアとしての経験は、ネットワークインフラの構築や運用において非常に貴重なものであり、 それを活かしつつ、クラウドエンジニアとしてのスキルを磨くことで、より幅広い視野で問題を解決できる自信があります。

    また、クラウドエンジニアの領域では、AI基盤を構築するための最新の技術に触れる機会が増えると考えています。 これにより、自らの成長に繋がるだけでなく、企業にとっても価値のあるエンジニアとなれると信じています。

    新しい分野への転職にはリスクもあるかもしれませんが、私は挑戦を恐れず、変化を受け入れる柔軟性があると自負しています。 クラウドエンジニアとしてのキャリアを積むことで、より多くの経験を得て、自らの可能性を広げることを熱望しています。

    弊社に応募した理由は何ですか？当社のどの点が魅力的だと思いますか？#

    ネットワークだけではなく、インフラ領域の様々スキルと知識を身につけることができ、それを活かす事ができる点に魅力を感じております。

    エンジニアは技術を使って効率化する事であったり、新たな価値を生み出すことで、より価値があるものと考えております。 今の現場で派遣エンジニアの立場では、ネットワークの高度な技術が求められますが、エンジニアは案件をまわすだけの役割しか与えられず、 目の前の案件をこなすだけの状態に危機感を覚え、いずれエンジニアとしての価値がなくなると感じました。

    また、検証自動化の推進を提案しても、自動化に使用する技術を知らないがために、技術を使って取り組んでいることがいか素晴らしいことであっても、 判断できる人がいないため、見送られた経験がありました。

    この経験からエンジニア自身が技術を使ってビジネスを推進していけることが重要だと思うようになり、将来は学んだ技術を活かしてプロジェクトを推進するPMを目指しております。

    将来のキャリアビジョンを教えてください#

    自らの知識と経験を活かして、将来はPMとしての立場で働くことをを目指しております。

    志したきっかけですが、炎上したり現場が疲弊しているプロジェクトに携わった経験からです。 実際に手を動かす人がうまく仕事をしていれば、プロジェクトは円滑に進むであろうと考えておりましたが、実際はそうではないことを経験しました。 私が携わったあるプロジェクトでは、最技術的な難しさに配慮が行き届かず工数の見積もりが甘かったり、PMの方が技術に関しては現場に任せっきりであったために現場との意思疎通がうまくいかず、 その結果、想定外の問題（ある機能が未サポートであったこと）を事前に見つけることができずに、プロジェクトの後半で問題に対処していたために納期が遅れるといった状況に遭遇しました。

    この経験で苦労したので、技術に対する理解とそれに基づく正しい意思決定を行い、プロジェクトを推進する立場として働きたいと思い、 PMの職を志すようになりました。

    ★要修正 また、技術力はあってもコミュニケ－ション能力が人よりない人が～ 誰もがチームで成果を出すための～ 自らの貢献により組織の利益に寄与したい

    マラソンの経験～ スポーツの経験から～

    プロジェクトの全体像を把握し、成果を出す: プロジェクトマネージャーはプロジェクトの全体像を理解し、 リソースやスケジュールを管理しながら、成果を出す役割を果たします。結果を生み出す裏方としての役割に魅力を感じるかもしれません。

    チームの協力と協力者としての役割: ITプロジェクトマネージャーは異なる部門やステークホルダーと協力し、 チームを統率して目標に向かって進める役割を担います。チームをまとめることで、多様な人材と協力して成果を上げたいという希望があるかもしれません。

    技術的な挑戦と問題解決: プロジェクトマネージャーはプロジェクトの進行に伴うさまざまな課題に直面します。それらの問題を解決するために技術的な知識と洞察力を活用し、プロジェクトを成功に導くことに興味があるかもしれません。

    プロジェクト全体の戦略とビジョンの実現: ITプロジェクトマネージャーはビジネスの戦略や目標を理解し、それを実現するためのプロジェクトを計画・実行します。自分の貢献が組織の大局に寄与するという意義を感じるかもしれません。

    変化と成長の機会: プロジェクトマネージャーは様々なプロジェクトに関わることで、新しい技術や業務に触れる機会があります。自己成長と学び続けることに興味があるかもしれません。

    IT 業界に転職した理由は？#

    飲食店の仕事では繰り返し行われる仕事が大半を占め、効率化を極めたとしても、人がやらなければならばならない仕事があるため、限界を感じておりました。 IT エンジニアの仕事は繰り返し行われる業務をアイディアで効率化、もしくは人の手を介さずに自動化できるとこが、飲食の仕事にはない魅力でした。 自分自身のスキルアップによって業務を改善したり、新たな価値を生み出すことに魅力を感じたことをきっかけに、ITエンジニアとして働きたいと考えて 転職いたしました。

    ネットワークを選んだ理由としては、当時未経験からの転職で門戸が開かれていたのが、ネットワークだったというの理由の1つですが、 通信インフラは日常生活に欠かせないモノであるため、社会的に重要な仕事で安定もしているだろうと考えたためネットワークの道に進みました。

    得意分野について教えてください。#

    得意とする分野はネットワークに関する技術とPython を用いた業務の効率化です。

    ネットワークに関しては 業務で通信キャリア様のネットワークの検証業務に携わっておりましたので、 OSPF / BGP MPLSのルーティング技術については実際に手を動かしながら理解を深めて参りました。 また、自宅にラボ環境を構築し、業務で分からなかった点は補う努力をしてきました。

    Python を用いたネットワーク自動化と業務強みがあります。 ネットワーク自動化ではそれまで人が機器にログインして行っていた作業を python (pyATS)へ置き換えて、 工数削減に削減に取り組んだ経験があります。

    また、数万行のコンフィグをpython を使用して、パラメーターを基に機械的にコンフィグを作成する簡易的な仕組みを作ったり、 パラメーターから構成図を自動的に作成する仕組みを作ったりするなどに取り組んでおりましたので、そうした経験からpython を用いて 業務を効率化することに強みがあります。

    クラウドエンジニアに転職を希望する理由は何ですか？なぜこの分野に興味を持ちましたか？#

    基盤にクラウドを用いており、クラウドではオンプレにありがちな単純な作業を効率化できる仕組みが用意されており、 その技術を用いてより付加価値の高い業務に集中できる点に魅力を感じたため、興味を持ちました
