    LPIC レベル1 スピードマスター 101

    1章　基本コマンドとファイルの操作

    1-1

    シェル変数の一覧を表示するには、どのコマンドラインを実行すればよいか？

    • set


    1-2

    現在の作業ディレクトリパスを表示するコマンドは？

    • echo ${PWD}
    • pwd


    1-3

    現在のシェルに、パラメータやオプションを設定するコマンドは？

    • set

    Info

    set コマンドはオプションを指定せずに実行すると、シェル変数、シェル関数を一覧表示します。 オプションを指定すると、シェルのパラメータなどを設定できます。


    1-4

    export コマンドの目的として適切なものは？

    • シェル変数を環境変数に設定し、bashから起動するアプリケーションが参照できるようにする


    1-5

    bash シェルを使用している場合、PS1='[\u@\h \w]'の設定の意味で適切なものは？

    • bash シェルのコマンドプロンプトを設定している


    1-6

    cd ~yukoコマンドを実行した際、適切なモノは？

    • ユーザyukoのホームディレクトリに移動する


    1-7

    manコマンドが実行時に参照するドキュメントファイルの置かれている場所として、適切なものは？

    • /usr/share/man
    • /usr/local/share/man


    1-8

    あるコマンドの第1セクションのマニュアルを表示するコマンドとして適切なモノは？

    • man 1 command

    Info

    特定のセクションを指定する場合はman コマンドの後に数字でセクションを指定します。


    1-9

    コマンドの実行履歴が保存されているファイルとして、適切なモノは

    • ~/.bash_history

    Tip

    コマンドの実行履歴はhistoryコマンドで確認できます。


    1-10

    あなたは現在、一般ユーザであるyukoとしてシステムにログインしています。しかし都合により、 現在のシェルを終了することなくroot権限と環境でログインし直す必要があります。適切なコマンドラインは？

    • su - root


    1-11

    ユーザyukoは自分のホームディレクトリ下にdir_b/subdirを作成したいが、dir_bは存在していません。 dir_b/subdirを作るために適切なものは？

    • mkdir -p /home/yuko/dir_b/subdir

    Tip

    mkdir コマンドのオプション-pを指定するとパス途中のディレクトリも作成することができます。


    1-12

    /tmpAディレクトリの中にあるすべてのファイル・サブディレクトリをカレントディレクトリに移動させたい場合、適切なコマンドラインは？

    • mv -f /tmp/A/*　.

    Info

    -f オプションは移動先に同名ファイルが存在しても強制的に上書きします。


    1-13

    /usr/local/samples ディレクトリ内にあるすべてのファイル・サブディレクトリを/home/yuko/samplesディレクトリの 下にコピーしたい場合、適切なものは？

    • cp -Rf /usr/local/sample/* /home/yuko/samples

    Info

    cp コマンドではディレクトリのコピーを行う場合は、-R（もしくは-r）オプションが必要です。


    1-14

    /tmpディレクトリの下にあるサブディレクトリをすべて削除するために、適切なものは？

    • rm -rf /tmp/*

    Info

    rm コマンドでディレクトリごと削除を行う場合は、-R（もしくは-r）オプションが必要です。


    1-15

    圧縮されたtarファイルであるfile.tar.gzに格納されているファイル名の一覧を標示するために、適切なコマンドラインは？

    • tar ztf file.tar.gz
    option description
    -c アーカイブファイルを作成する
    -t アーカイブファイルの内容を表示する
    -x アーカイブファイルを展開する
    -f アーカイブファイル名を指定する
    -v 詳細情報を表示する
    -j アーカイブファイルに対して、bzip2を使用する
    -J アーカイブファイルに対して、xzを使用する
    -z アーカイブファイルに対して、gzipを使用する

    Info

    圧縮されたテキストファイルを展開せずに、ファイルの内容を標準出力するコマンドとして、zcat(gzip用)、bzcat(bzip2用)、xzcat(xz用)があります


    1-16

    カレントディレクトリにあるsamples.tgzには、z.txtとb.txt ファイルが格納されている。「gunzip samples.tgz」と 実行した際に、カレントディレクトリにあるファイルはどれですか？

    • samples.tar


    1-17

    表示が1画面に入りきらない場合に、1画面ずつ表示するコマンドとして適切なモノは？

    • more

    Tip

    1画面に収まらないファイルを表示するコマンドとしてless コマンドもあります


    1-18

    ファイルのタイプを確認するコマンドは？

    • file


    1-19

    bash を使用している場合に、コマンドの後で"1>&2"と指定した場合の説明は？

    • 標準出力（stdout）を標準エラー出力（stderr）にリダイレクトする

    Info

    • ファイル記述子「0」：標準入力
    • ファイル記述子「1」：標準出力
    • ファイル記述子「2」：標準エラー出力

    Tip

    • 例1：カレントディレクトリのファイルリストをfile1 に格納する
      $ ls > file1
    • 例2：1>を使用してファイルリストをfile2に格納する（例1と同意）
      $ ls 1> file1
    • 例3：file1に、/binのファイルリストを追記して保存する
      $ ls > file1
    • 例4：ls コマンドを実行してエラー出力された場合のみfile3ファイルに格納する
      $ ls 存在しないファイル 存在するファイル 2> file3
    • 例5：標準出力、標準エラー出力の両方をbothファイルに格納する
      $ ls 存在しないファイル 存在するファイル &>both
      以下でも同様の結果を得られる
      $ ls 存在しないファイル 存在するファイル >& both **
      **$ ls 存在しないファイル 存在するファイル 1> both 2>&1
    • 例6：コマンド1を実行した結果の標準出力、標準エラー出力の両方をbothファイルに格納する
      $ コマンド1 &> both
      以下でも同様の結果を得られる $ コマンド1 >& both


    1-20

    コマンド「cmd 1> data」を実行した場合の説明で正しいものは？

    • コマンドcmdの標準出力をdataに格納する

    Info

    「cmd > data」と同意です。

    Note

    コマンドを実行した結果の標準出力、標準エラー出力の両方をファイルに格納するには ls >& test.txtのように記述します。


    1-21

    カレントディレクトリにfileAがない場合に、ls > fileAと実行した際の説明として適切なモノはどれか？

    • fileAが新規に作成され、lsの標準出力結果が書き込まれる


    1-22

    以下のように3行の長文をcatコマンドへの標準入力として取り込んで画面に表示します。空欄に入れる2文字の記号を記述せよ。 なお、この例ではcatのコマンドラインの最後に指定した「END」を入力終了を指示する文字列とします。 

    $ cat ______ END
> Free software is amatter of liberty, not price.
> To understand the concept, you should think of free
> as in free speech, not as in free beer.
END

    • <<

    Info

    「<<」はヒアドキュメント（here document）と呼び、主にシェルスクリプトの中で利用されます。 cat <<eosのように「<<文字列」を指定すると、指定した文字列が現れるまでの内容を、コマンドへの標準入力として扱います。 この文字列はdelimiter、separatorあるいは識別文字列などとも呼ばれます。


    1-23

    あるプログラムの実行結果である標準出力、標準エラー出力のどちらもあらゆる端末、ファイル、デバイスに表示させないようにする際、 そのプログラムのコマンドラインの最後の部分として適切なものはどれか？

    • > /dev/null 2>&1

    Tip

    /dev/null ファイルは特殊なデバイスファイルで、このファイルに書かれたものはすべて消去され、このファイルを読み込むと常にEOFが返ります。


    1-24

    コマンドの標準出力を別のコマンドにリダイレクトすると同時にファイルにも格納することのできるコマンドを記述せよ。

    • tee

    Info

    teeコマンドは、標準入力から読み込んだデータを標準出力とファイルの両方に出力します。


    1-25

    コマンドcmd1を実行後、コマンドcmd2を実行する際、適切なものは？

    • cmd1 ; cmd2

    Tip

    コマンドを連続実行する際に「;」「&&」「||」を使用します。

    • 例）コマンド1、コマンド2を順々に実行する

      • コマンド1 ; コマンド2

    • 例）コマンド1が実行できれば、次のコマンド2を実行する

      • コマンド1 && コマンド2

    • 例）コマンド1が実行できなければ、次のコマンド2を実行する

      • コマンド1 || コマンド2


    1-26

    file1 から文字列fooを大文字/小文字を区別せずに検索するコマンドラインとして、適切なモノは？

    • cat file1 | grep -i foo
    option description
    -v パターンに一致しない行を表示する
    -n 行番号を表示する
    -l パターンと一致するファイル名を表示
    *-i * 大文字と小文字を区別しないで検索を行う


    1-27

    ファイルの先頭から、任意の行数を指定して、その内容を表示するコマンドを記述せよ。

    Info

    • 【指示1】すべてのユーザが読み取り及び実行可能
      • a（すべてのユーザ）に　x （実行権）　+（追加する）
    • 【指示2】所有者のみは書き込みも可能
      • 現在、所有者は　w（書き込み）が付与されているため、変更しない。
      • ただし、グループに付与されている　w（書き込み）を　-　（削除）する
    • 上記を合わせるとa+x,g-w　にて指定して権限変更（オクタルモードでは755を指定）
    パーミッション 数値
    読み取り権 4
    書き込み権 2
    実行権 1
    権限なし 0
    権限 数値 意味
    rwx 7 読込み・書き込み・実行権あり
    rw- 6 読み取り・書き込み権あり
    r-x 5 読み取り・実行権あり
    r-- 4 読み取りのみ
    --- 0 権限なし

      記号 意味
      *u * 所有者
      g グループ
      o その他
      a すべてのユーザ
      • ②操作
      記号 意味

      - 許可を削除する

      • ③パーミッション
      記号 意味

      • リンクファイルが使用するiノード番号は同じ番号
      • ディレクトリを基にリンクファイルを作成することはできない
      • iノード番号は同一ファイルシステム内でユニークな番号なので、異なるパーティションのハードリンクを作成することはできない
    • シンボリックリンク：ln　-s　オリジナルファイル名　リンク名

    6-1

    Info

    systemd のターゲット（target）の説明で誤っているいるものはどれか？1つ選択

    • ターゲットはsystemd の管理対象であるユニットタイプの1つである
    • ブートローダのカーネルオプションでデフォルトターゲットを変更する場合は、「systemd.target=」の後にターゲットを指定する
    • デフォルトランレベルに相当するターゲットはdefault.targetである
    • ターゲットはSysV initのランレベルに相当する

    Info

    正しくはデフォルトターゲットを変更する場合はsystemctl コマンドで ターゲットユニット （.target ファイル拡張子）を指定します。 なお、ターゲット名は .target を省略することができます

    Info

    systemdはSysVinitを置き換える新しいinitの仕組みです。systemdでは以下のデーモンプロセスが連携して動作しています。

    プロセス名 機能
    systemd systemd の本体となるプロセスで、initの代わりに常駐します
    systemd-journald systemdが制御する、各サービスが出力するログを扱います。
    systemd-logind システムへのログイン、起動停止、電源ボタン操作などを管理します。
    systemd-udevd 従来のudevdに代わり、デバイスの動的変更、検出を行います。

    systemdで扱う処理はUnitという単位で管理します。Unitは設定ファイルであり、Unitの設定に従ってsystemd自体が処理を実行します。SysVinitのようにスクリプトを実行するわけではありません。

    Unitには各機能ごとに拡張子が割り当てられており、拡張子を見ることでどういった機能のためのUnitかが判別できるようになっています。

    拡張子 機能
    device 各種デバイスを管理するUnit
    mount ファイルシステムのマウントを管理するunit
    service サービスを制御するUnit
    swap スワップ領域を管理するUnit
    target 複数のサービスを一つのグループにするためのUnit

    また、SysVinitではプロセスをPIDによって管理していましたが、systemdではcgroupsという Linuxカーネルの機能によってプロセスのリソースを管理できます。

    systemdはUpstartと同様、各サービスを並列起動することができます。そのため、順次起動していく SysVinitに比べて高速なシステム起動や停止が行えます。 システム起動時に最初に実行されるUnitは「/etc/systemd/system/default.target」です。 「default.target」をSysVinitでのランレベルに相当するUnitへのシンボリックリンクとして作成することで、 期待するサービス群を起動できるようになります。

    シンボリックリンクとは、Windowsでのショートカットのようなもので、元ファイルの場所を指し示すリンクの事です。 シンボリックリンクが持っている情報は「元ファイル（ディレクトリ）がどこにあるのか」というパス情報のみです。 シンボリックリンクを作成するにはlnコマンドに「-s」オプションを付加して作成します。 （書式：ln -s 元ファイル リンクファイル）

    SysVinitのランレベルとsystemdのターゲットは以下のように対応しています。

    ランレベル ターゲット 意味
    0 poweroff.target システムをシャットダウンし、電源を切ります。
    1 rescue.target レスキューシェルを設定します。
    2,3,4 multi-user.target 非グラフィカルなマルチユーザーシステムを設定します。
    5 graphical.target グラフィカルなマルチユーザーシステムを設定します。
    6 *reboot.traget * システムをシャットダウンして再起動します。

    参考：SysV ランレベルと systemd ターゲットの比較

    これらのターゲットは「/lib/systemd/system」配下に格納されています。 また、SysVinitとの関連がわかりやすいように、runlevel0.target〜runlevel6.targetまでの シンボリックリンクも用意されています。

    • 所有者は読み取り書き込み実行ができ、所有グループは読み取り、実行ができ、その他のユーザはアクセスが禁止されるようなumask値は0027

    • コマンドの完全パスを確認できるのはwhichコマンドloacateコマンドです（whereis はマニュアルの場所）

    • 電源管理デーモンとして起動時に実行されるもの：　/usr/sbin/acpid
